# คลังเรฟเฟอเรนซ์ (Reference Collector)

เว็บแอพเก็บลิงก์เรฟเฟอเรนซ์ วางลิงก์แล้วระบบจัดหมวดให้อัตโนมัติ

รองรับ 2 โหมด:
- **ไม่ตั้งค่าอะไรเลย** → เก็บข้อมูลใน `localStorage` ของเบราว์เซอร์ (เครื่องเดียว)
- **ตั้งค่า Supabase** → ข้อมูลซิงค์ข้ามอุปกรณ์ได้จริง (แนะนำ)

---

## ขั้นตอนที่ 1: สร้างฐานข้อมูล Supabase (ฟรี)

1. สมัคร/ล็อกอินที่ [supabase.com](https://supabase.com) (ใช้ GitHub ล็อกอินได้)
2. กด **New Project** → ตั้งชื่อโปรเจกต์ (อะไรก็ได้) → ตั้งรหัสผ่าน database (เก็บไว้ ไม่ต้องใช้ในขั้นตอนนี้) → เลือก region ใกล้ๆ (เช่น Singapore) → กด **Create new project**
3. รอสัก 1-2 นาทีให้โปรเจกต์สร้างเสร็จ
4. ไปที่เมนู **SQL Editor** (ไอคอนรูปฐานข้อมูลด้านซ้าย) → **New query**
5. เปิดไฟล์ `supabase/schema.sql` ในโฟลเดอร์นี้ คัดลอกทั้งหมดไปวาง แล้วกด **Run**
   → จะได้ตาราง `links` พร้อมใช้งาน
6. ไปที่ **Project Settings** (ไอคอนเฟือง) → **API**
   → คัดลอกค่า 2 อย่าง: **Project URL** และ **anon public key**

## ขั้นตอนที่ 2: ใส่ค่าลงโปรเจกต์

1. คัดลอกไฟล์ `.env.example` แล้วเปลี่ยนชื่อเป็น `.env`
2. ใส่ค่าที่คัดลอกมาจาก Supabase:
   ```
   VITE_SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-public-key
   ```

## รันบนเครื่องตัวเอง (ทดสอบก่อน deploy)

ต้องมี [Node.js](https://nodejs.org) เวอร์ชัน 18 ขึ้นไป

```bash
npm install
npm run dev
```

เปิดที่ `http://localhost:5173` — ถ้าตั้งค่า Supabase ถูกต้อง แถบเตือนสีเหลือง "ยังไม่ได้ต่อ Supabase" จะหายไป และลิงก์ที่เพิ่มจะไปโผล่ใน Supabase Table Editor ด้วย

## ขั้นตอนที่ 3: Deploy ขึ้นเว็บจริงด้วย Vercel

1. อัปโหลดโฟลเดอร์นี้ขึ้น GitHub repo ใหม่ (**อย่าลืม**: ไฟล์ `.env` จะไม่ถูกอัปโหลดเพราะอยู่ใน `.gitignore` แล้ว — เป็นเรื่องปกติและปลอดภัย)
2. ไปที่ [vercel.com](https://vercel.com) → **Add New Project** → เลือก repo นี้
3. Vercel จะตรวจพบว่าเป็น Vite ให้อัตโนมัติ **ก่อนกด Deploy** ให้เปิดส่วน **Environment Variables** แล้วใส่ 2 ตัวแปรเดียวกับใน `.env`:
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. กด **Deploy** รอสักครู่ จะได้ลิงก์เว็บ เช่น `https://your-project.vercel.app`
5. เปิดจากมือถือหรือคอมเครื่องไหนก็ได้ ข้อมูลจะซิงค์กันหมดเพราะอยู่ใน Supabase แล้ว

### หรือใช้ Vercel CLI (ไม่ต้องใช้ GitHub)

```bash
npm install -g vercel
vercel
```

ตอบคำถามตามที่ CLI ถาม แล้วอย่าลืมตั้งค่า Environment Variables ผ่านคำสั่ง `vercel env add` หรือในหน้าเว็บ Vercel ทีหลัง

## ข้อควรรู้เรื่องความปลอดภัย

`anon key` ของ Supabase จะถูกฝังอยู่ในโค้ด JS ที่ส่งไปให้เบราว์เซอร์ผู้ใช้ (เป็นเรื่องปกติสำหรับแอพฝั่ง client) ตาราง `links` ที่สร้างให้เปิดอ่าน/เขียนได้จาก anon key โดยไม่ต้องล็อกอิน — เหมาะสำหรับใช้คนเดียวหรือแชร์กับทีมเล็กที่ไว้ใจกัน **ไม่เหมาะกับการแชร์ลิงก์เว็บให้คนแปลกหน้าเข้าถึง** เพราะใครก็แก้ไข/ลบข้อมูลได้ ถ้าต้องการระบบล็อกอินจริงจัง (เช่น Supabase Auth) บอกได้ ช่วยต่อให้ได้เพิ่ม

## แก้ไข/ปรับแต่ง

โค้ดหลักอยู่ที่ `src/App.jsx` ปรับสี, ฟอนต์, หรือกฎการจัดหมวดหมู่ (ตัวแปร `KNOWN_SITES` และ `KEYWORD_RULES`) ได้ตรงนั้น
การเชื่อมต่อฐานข้อมูลอยู่ที่ `src/lib/supabaseClient.js`
